# Instrucciones para estudiantes

Este examen evalúa Git Flow mediante cambios pequeños sobre una página HTML existente.

## Reglas generales

- Sigue los issues en orden.
- Modifica solo lo solicitado en la misión activa.
- No adelantes cambios de misiones futuras.
- No cierres issues manualmente.
- No modifiques el README para intentar completar el examen.

## Proyecto

El archivo de trabajo principal es `index.html`.

Cada misión te dirá:

- la rama esperada;
- el cambio exacto que debe aparecer en `index.html`;
- los cambios que todavía no están permitidos;
- el criterio automático de cierre.

## Revisión automática

El workflow **Validar progreso de misiones** revisa ramas, Pull Requests y contenido de `index.html`.

Si una misión cumple los criterios, el workflow cerrará el issue y creará la siguiente misión. Si falta algo, dejará una revisión breve en el issue.

## Página local

Para ver la página durante el examen, abre `index.html` directamente en el navegador. No necesitas instalar dependencias ni iniciar un servidor.
